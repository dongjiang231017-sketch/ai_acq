## 3. 今天新踩的坑（血泪，别重踩）

1. **DashScope 静默忽略 `conversation.item.create`（暗坑4）**：开场白想靠"塞一条 user 文本
   再 response.create"照稿播——该模型直接不理这条 item，只凭 session instructions 即兴发挥。
   **结论：想让开场白一字不差，唯一可靠解是固定录音（OpeningPlayer），不能靠模型。**
2. **`input_speech_started` emit 必须带 `llm.InputSpeechStartedEvent()`**：不带参数→框架回调
   `TypeError`→异常落在 `run_coroutine_threadsafe` 的 future 里被吞→interrupt() 永不执行→
   打断彻底失效。这类"静默吞异常"单看日志没有报错，极难查。
3. **`user_initiated=True` 会让 VAD 轮次回复被框架丢弃**：框架 `_on_generation_created` 对
   `user_initiated=True` 直接 return（由 _realtime_reply_task 消费 future）。VAD 触发的回复
   必须标 False，否则模型明明生成了音频（日志一堆 audio.delta）却永远不播 = "只说第一句"。
4. **`AudioSource.capture_frame` 不保证自阻塞**：把整段录音塞进去会瞬间返回（日志"开始/结束"
   同一毫秒）。必须自己按帧墙钟 `asyncio.sleep` 节奏推，play() 才在真实时长后返回。
5. **UC100 网关"提前应答"无法从网关侧关**：整个 wan_default SIP profile 翻遍了，固件没有
   early-media / 应答后接通 / PRACK 相关可用开关（PRACK 是关的）。网关收到呼叫立即回 200 OK，
   GSM 侧还在振铃时开场白就开播。**治本在 agent 侧：本地能量 VAD 等真人开口。**
6. **plus 模型不支持 Cherry 音色**：换 plus 必须同时换音色（用 Serena），否则报
   `Voice 'Cherry' is not supported`。换模型/音色后 `opening.wav` 必须用新组合重新生成。
7. **同号高频呼叫被手机/运营商拦截**：测试号 18107090349 当天打到第 4-5 通就"拨通但手机不响"
   （SIP 侧 200/话单有记录但 GSM 侧无来电）。这是风控前兆，别硬打。换被叫号轮着测。

## 4. 还没解决的问题（下一个人从这里接，按优先级）

1. **【话术】收网临门一脚会踢飞**：客户明确同意（"好好好，嗯"）时，应该问"您这号就是微信吧？"
   确认号码后再道别，但模型有时直接"不打扰您了，再见"跑路，放弃了最该拿下的时刻。
   改法：收网阶段提示词加硬约束"客户同意后必须先确认手机号是不是微信，才能道别"。
2. **【bug】意向标记偶尔标不上**：框架的 user 消息事件在文本转写完成前就发出，标记逻辑拿到空
   文本。应改用 `conversation.item.input_audio_transcription.completed` 事件（带 transcript）
   做标记判定，而不是 `conversation_item_added`。
3. **【红线】plus 模型偶尔擦边**：说过"我们是视频号**官方合作**服务商"（夸大）、编造"核销时
   系统自动弹出加微信入口"（实际是人工顺手加）。收网/知识提示词要加禁语清单。
4. **【开场】彩铃可能误触发本地 VAD**：本地 VAD 靠能量判断，若被叫号设了彩铃（振铃时放音乐/
   人声），可能被当成"真人开口"提前触发开场白。真机遇到再调 `OPENING_SPEAK_RMS` 阈值。
5. **【上量】线路必须换轨**：UC100/鼎信插 SIP 卡是行业封号率最高的形态。见
   `AI电销系统业界对标与升级方案.docx`（在用户 Downloads），要换 AXB 中间号/回拨线路。
6. **【自测基建】`test_multiturn.py` 偶发误报 FAIL**：音频段检测在录音轨道断流时段落闭合时机
   有边角问题，opening 段偶尔数不到。是测试脚本问题，不是产品 bug。判断产品好坏以真机为准。

## 5. 话术当前设计（想调话术改这里）

`bench_agent_reference.py`（≈ Workbuddy 的 `agent.py`）里：
- `BASE_RULES`：身份/说话铁律/参考知识/红线/结束条件（全程通用）
- `STAGE_PROMPTS`：`icebreak`（破冰，禁提微信）/ `value`（讲价值，不主动要微信）/
  `close`（收网，才解锁加微信话术）
- 阶段切换逻辑在 `_on_item`：客户出现意向关键词/赶时间 → 直接 `close`；破冰接住第一反应 →
  `value`；`value` 聊满 4 轮还没信号 → `close`。切换时 `update_instructions` 动态换提示词。
- 核心目标：**引导加微信**。电话不成交、不报价（问价统一"方案报价微信发您"）。

配套还有份人工话术手册 `视频号团购电销话术手册.docx`（用户 Downloads），一问一答实战版，
是话术的知识来源，可反哺提示词。
